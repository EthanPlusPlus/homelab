- **Decision 021 adopted — ReviewItems as the human-judgment boundary; synthesis-service is the next build** — consolidation decision naming Prismo's center of gravity. Core principle: operational signals are internal, human-facing work units are ReviewItems. End-to-end loop: signals → synthesis evaluates significance (quality gate enforced) → ReviewItem in review-queue → human approve/reject/edit → canon updated → doctrine validates post-approval. ReviewItem becomes the universal Layer 4 integration primitive — every surface (CLI, web, WhatsApp, mobile, ambient) is a thin renderer + action handler against the same contract. Retires proposed-idea 012 (superseded), deprecates `prismo capture promote`, freezes `drafts/` as legacy, resequences doctrine-service Day-1 build to post-synthesis (Decision 020 scope unchanged, only order). - **Decision 020 adopted — doctrine-service scoped as structural coherence engine** — locks the Phase 3 boundary before any code exists. Core principle: doctrine produces *measurements*, synthesis produces *judgments*. Core test: self-evident vs arguable (counts/ratios/distributions OK; weighted scores/health/urgency/importance forbidden). Day-1 capabilities: staleness eval, supersession-minimal, metadata reconciliation, provenance integrity (Decision 018), structural relationship topology. Out: summarization, narratives, semantic contradiction/drift, scores, prioritization-by-inferred-importance — all migrate to synthesis-service. Directionality invariant locked: synthesis may read doctrine freely, may NEVER write doctrine state. Module-first at top-level `doctrine/` (peer to `api/`, `workflow/`). Tests required, not aspirational. Decision draws the structural-memory-vs-interpretation boundary that most org tools silently merge — the architecture's strongest emergent property.
