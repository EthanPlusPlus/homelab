# Capability Contracts

Defines organizational capabilities as stable, provider-independent interfaces. These contracts sit between Layer 2 (operational services) and Layer 3 (runtime intelligence). Any runtime that implements a contract can be swapped in without changing the calling layer.

The contract is the interface. The runtime is the implementation. Prismo code depends on contracts, not providers.

---

## Contract Definitions

---

### retrieve

Semantic search over canon and code.

```typescript
retrieve(
  query: string,
  options: {
    project?: string,        // scope to a specific project
    category?: string,       // scope to a doc category (decisions, architecture, etc.)
    lifecycle?: string[],    // filter by status: ["active", "proposed"]
    top_k?: number,
    include_metadata?: boolean
  }
) → RetrievalResult[]
```

**Provider requirement:** embedding model + vector store
**Current implementation:** context-server MCP (ChromaDB + sentence-transformers)
**V2 implementation:** context-server v2 with metadata-aware filtering

---

### summarize

Condense content into a shorter form while preserving key information.

```typescript
summarize(
  content: string | string[],
  options: {
    target_length?: "short" | "medium" | "long",
    format?: "prose" | "bullets",
    focus?: string           // what aspect to emphasize
  }
) → string
```

**Provider requirement:** language model
**Suitable runtimes:** local 7B+ model, hosted API

---

### synthesize

Combine multiple sources into a coherent unified artifact.

```typescript
synthesize(
  sources: RetrievalResult[],
  options: {
    output_type: "brief" | "summary" | "onboarding" | "snapshot",
    project?: string,
    audience?: "technical" | "general"
  }
) → string
```

**Provider requirement:** language model with large context
**Suitable runtimes:** local 70B model, hosted API

---

### codegen

Generate or modify code from a specification.

```typescript
codegen(
  spec: string,
  context: RetrievalResult[],
  options: {
    language?: string,
    style?: string,
    mode?: "generate" | "modify" | "review"
  }
) → string
```

**Provider requirement:** language model with strong code capability
**Suitable runtimes:** Claude, GPT-4o, local coding model (DeepSeek Coder, Qwen2.5-Coder)
**Note:** This is the primary capability Ethan uses via Claude Code. Highest quality bar.

---

### plan

Decompose a goal into ordered steps with dependencies.

```typescript
plan(
  goal: string,
  context: RetrievalResult[],
  options: {
    granularity?: "high" | "medium" | "low",
    format?: "sequential" | "tree"
  }
) → PlanStep[]
```

**Provider requirement:** language model
**Suitable runtimes:** local 70B model, hosted API

---

### hydrateSession

Generate a ContextBundle for a session start. Replaces the warm-up conversation ritual.

```typescript
hydrateSession(
  contributor: string,
  project: string,
  options: {
    depth?: "minimal" | "standard" | "full"
  }
) → ContextBundle
```

**Provider requirement:** context-server v2 (Layer 2 service — not a language model call)
**This is a service capability, not a model capability.** The ContextBundle is assembled from structured canon state, not generated by a model.

---

### generateOperationalBrief

Produce a human-readable summary of organizational state for a project.

```typescript
generateOperationalBrief(
  project: string,
  options: {
    audience?: "ethan" | "shrey" | "kyle" | "new-contributor"
  }
) → OperationalBrief
```

**Provider requirement:** context-server v2 (retrieval) + language model (synthesis)
**Hybrid capability:** retrieval is structural; prose generation uses a model.

---

### captureSignal

Extract structured canon candidates from conversational input.

```typescript
captureSignal(
  input: string,
  context: {
    project: string,
    contributor: string,
    session_id: string
  }
) → SignalCandidate[]

// SignalCandidate:
{
  type: "decision" | "proposal" | "tension" | "action",
  content: string,
  confidence: number,
  suggested_title: string
}
```

**Provider requirement:** language model
**Human review required before any candidate becomes canon.** This capability drafts; humans approve.

---

### detectContradictions

Surface conflicts between canon documents.

```typescript
detectContradictions(
  docs: CanonDoc[],
  options: {
    project?: string,
    categories?: string[]
  }
) → Contradiction[]
```

**Provider requirement:** language model + structured comparison
**Primary consumer:** Sukuna v2

---

### detectStaleness

Identify canon documents that may be outdated.

```typescript
detectStaleness(
  docs: CanonDoc[],
  rules: StalenessRule[]
) → StaleItem[]
```

**Provider requirement:** doctrine-service (rule-based — no model required for basic detection)
**Model-assisted for nuanced cases** (e.g., "is this decision still consistent with V2 direction?")

---

## Contract Stability Rules

1. Contract names and input signatures are stable once adopted
2. Response shapes may evolve with versioning (v1, v2)
3. Options fields are always optional — adding new options is non-breaking
4. A runtime that cannot implement a capability must declare it unsupported in its RuntimeProfile
5. The routing engine selects the best available runtime per capability per request
