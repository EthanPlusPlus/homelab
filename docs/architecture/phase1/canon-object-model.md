# Canon Object Model

Defines the standard metadata schema for all canonical objects in Prismo V2. The goal is lightweight, consistent structure that enables lifecycle management, retrieval filtering, and relationship traversal — without over-engineering the format.

Existing V1 docs do not need immediate migration. This schema is the target state for new docs and for progressive updates as V2 services are built.

---

## Markdown-Resident Objects

These live as markdown files in `~/canon/`. YAML frontmatter carries structured metadata; body carries human-readable content.

---

### Decision

```yaml
id: "013"
title: "Short descriptive title"
type: decision
status: active | superseded | archived
created_at: "2026-05-12"
updated_at: "2026-05-12"
author: ethan
project: homelab | context-server | even | flight-planner | prismo-v2
supersedes: "012"         # optional — id of decision this replaces
superseded_by: "014"      # optional — filled when this is superseded
tags: []
```

**Required:** id, title, type, status, created_at, project
**Optional:** author, supersedes, superseded_by, tags, updated_at

---

### Proposal

```yaml
id: "010"
title: "Short descriptive title"
type: proposal
status: proposed | experimental | active | superseded | closed | archived
created_at: "2026-05-12"
updated_at: "2026-05-12"
author: ethan
project: homelab
v2_program: "2"           # optional — which V2 program this maps to
tags: []
```

**Required:** id, title, type, status, created_at, project
**Optional:** author, v2_program, tags

### Project

```yaml
name: even
display_name: "Even"
type: project
status: active | archived
created_at: "2026-04-01"
description: "iOS receipt-splitting app"
repo: "github.com/EthanPlusPlus/even"
contributors: [ethan, shrey, kyle]
canon_path: "~/canon/even"
code_path: "~/projects/even"
```

**Required:** name, display_name, type, status, created_at, description
**Optional:** repo, contributors, canon_path, code_path

### Contributor

id: ethan

type: contributor

**Required:** id, display_name, type, role
**Optional:** technical_level, primary_interface, joined_at

## Runtime Objects (Data Shapes)

These are not markdown files. They are structured payloads produced and consumed by Layer 2 services. Defined here as schemas.

---

### ContextBundle

Produced by the Session Hydration System. Injected into runtime at session start.

```json
{
  "project": "even",
  "contributor": "ethan",
  "generated_at": "2026-05-12T10:00:00Z",
  "active_doctrine": [],
  "recent_changes": [],
  "active_workstreams": [],
  "unresolved_tensions": [],
  "related_history": [],
  "contributor_context": {}
}
```

---

### OperationalBrief

Generated by the Operational Brief Engine. Human-readable summary of organizational state.

```json
{
  "project": "even",
  "generated_at": "2026-05-12T10:00:00Z",
  "current_direction": "",
  "active_decisions": [],
  "unresolved_issues": [],
  "recent_evolution": [],
  "recommended_focus": []
}
```

---

### RuntimeProfile

Describes a registered model provider and its capabilities.

```json
{
  "id": "claude-sonnet",
  "provider": "anthropic",
  "type": "hosted",
  "capabilities": ["retrieve", "summarize", "codegen", "synthesize", "plan"],
  "context_limit": 200000,
  "latency_tier": "medium",
  "cost_tier": "high",
  "routing_tags": ["coding", "complex-reasoning"]
}
```

---

### Workstream

Tracks an active unit of work.

```json
{
  "id": "ws-001",
  "title": "context-server v2 retrieval pipeline",
  "project": "context-server",
  "status": "active | paused | completed",
  "contributor": "ethan",
  "created_at": "2026-05-12",
  "last_active": "2026-05-12",
  "related_decisions": [],
  "related_proposals": []
}
```

---

## Schema Evolution Rules

1. Add fields as optional — never remove or rename existing required fields without a decision
2. Status field is always required — no status-less canon objects in V2
3. `type` field is required — enables cross-object retrieval filtering
4. `project` field is required — enables project-scoped queries
5. Frontmatter must be valid YAML — enforce in Sukuna v2's metadata reconciliation pass
