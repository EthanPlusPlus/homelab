## 2. Observations

- **Layer 1 instructions have not caught up to the Layer 2/3 architecture for the second time in a month.** Sukuna 2026-05-15 flagged that workflow-state-service shipped but `memory/mcp.md` and `memory/workflow.md` still described the manual warm-up. That was fixed. Now Decision 021 has shipped a full synthesis → ReviewItem → approval loop, and the same files still describe `prismo capture promote` and `drafts/` as live. This is not a one-off — it's the recurring failure mode: the Service Rule enforces HTTP-side contract truth, but nothing enforces that human-facing instructions stay aligned with the contract. Candidate for a Layer 1 doc-currency check: a Service Rule for memory/.
- **Decision 022 doesn't follow the canon decision format, and it was produced by the loop that's supposed to keep canon coherent.** No frontmatter, missing `Status`/`Date`/`Context`/`Consequences`, divergent title style. The first synthesis-service-emitted decision violates Decision 020's "metadata reconciliation: missing lifecycle fields, malformed frontmatter, invalid statuses." The decision approval ritual approved the content, not the structure. Doctrine-service hasn't shipped, so there's no automated check; the human-judgment boundary at `prismo review` didn't catch it either. Synthesis-service's prompt template is silently teaching the model "Prismo decisions look like this" using whichever shape the LLM defaulted to.
- **The "Retired" block in v2-progress.md (Phase 3) lists synthesis-service as retired** while elsewhere on the same page synthesis-service is listed as shipped (step 2). A direct contradiction inside the most-read living-index document. Likely a leftover from an earlier scoping doc that wasn't fully rewritten when Decision 021 landed. Same pass that retired Sukuna v2 accidentally also retired the thing it was promoting.
- **Sukuna v2 references persist in 7+ files** despite v2-progress.md retiring the name. Decision 013 (the masterplan-adoption record) still lists Sukuna v2 as a Phase 3 service in its open questions. Phase 1 contract docs (capability-contracts, canon-object-model, lifecycle-semantics, design-notes) describe Sukuna v2 as the primary consumer of multiple capabilities. The retirement was named in one place and unenforced everywhere else.
- **Four canon-shaping decisions in 48 hours, the fourth of which was synthesis-emitted and human-approved through a system that was shipped the same day.** Decision 019 annotation (2026-05-16) → Decision 020 (2026-05-16) → Decision 021 (2026-05-17) → Decision 022 (2026-05-17, auto-synthesized, approved via `prismo review`). There is no observation period between "the loop ships" and "the loop emits decisions into canon." The empirical confidence threshold the system is supposed to learn cannot be learned over hours. Decision 021's own "Observation week" precondition for doctrine-service is being skipped on the only thing currently writing to canon.
- **The 6 promoted captures in `drafts/` are now triple-orphaned.** Promoted manually with the intent of being authored into canon; tagged `record_type=draft` so excluded from retrieval; Decision 021 says "leave them where they are." Synthesis-service inputs are captures (operational state), not drafts. doctrine-service won't see them. No human authors them because the authoring step is the thing Decision 021 retired. Result: 6 rich signals stuck mid-lifecycle. The Decision 021 Consequences section frankly labels this as "stay in `drafts/`. Author them into canon manually if desired" — which is the workflow Decision 021 was written to eliminate.
- **proposed-idea 008 (agentic-decision-pipeline) and 011 (self-learning feedback loop) are now substantively superseded by Decision 021** but neither was marked superseded in the same pass that retired proposed-idea 012. 008 explicitly described "an agentic system that triages captures into decisions" — that's the synthesis-service. 011 described "Sukuna's output volume reaching review-bottleneck scale, then reconsider auto-approval" — Decision 021 explicitly defers auto-approval tiers to post-observation, which is the same conclusion. Both should be retired by reference.
